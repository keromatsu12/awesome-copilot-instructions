# 開発標準ガイドライン & Copilot指示書

[**English**](./README.md)

## 概要

本リポジトリは、プロジェクトで遵守すべき開発標準、アーキテクチャガイドライン、およびGitHub Copilotへの指示書を定義しています。コード品質、一貫性、拡張性を維持するための包括的なリファレンスとして機能します。

開発者およびAIアシスタント（特にGitHub Copilot）がこれらのガイドラインに従うことで、プロジェクトの核となる原則に沿った作業が可能になります。

## ディレクトリ構成

リポジトリは、一般的な指示と具体的な技術スキルを明確に分離して構成されています。

### `.github/`

このディレクトリには、開発ワークフローの中核となる設定とドキュメントが含まれています。

- **`copilot-instructions.md`**
  - **説明:** GitHub Copilot向けにカスタマイズされた指示書セットです。
  - **目的:** AIが生成するコードが、プロジェクト固有のコーディングスタイル、アーキテクチャパターン（クリーンアーキテクチャ）、およびテスト要件に準拠することを保証します。詳細なスキルガイドを参照する高レベルのスケルトンとして機能します。

- **`skills/`**
  - **説明:** 技術ドメインごとに分類された詳細なガイドと標準のコレクションです。各ディレクトリには、そのドメインを詳細に説明する `SKILL.md` ファイルが含まれています。
  - **内容:**
    - **`architecture/`**: クリーンアーキテクチャのレイヤー（ドメイン、アプリケーション、プレゼンテーション、インフラストラクチャ）と依存関係のルールを定義します。
    - **`backend-integration/`**: API統合、認証処理、データ同期に関するガイドラインです。
    - **`coding-standards/`**: 命名規則、コードフォーマット、およびベストプラクティス（SOLID, DRY, KISS）を強制します。
    - **`state-management/`**: グローバルおよびローカルの状態管理（例：Zustand, TanStack Query）の標準です。
    - **`testing/`**: ユニットテストおよび統合テストの戦略（AAAパターン、カバレッジ要件を含む）です。
    - **`ui-design/`**: レスポンシブUIの構築、スタイリング規約（CSS/BEM）、およびコンポーネント構造に関するガイドラインです。

## 利用方法

1.  **開発者:** 実装を開始する前に、担当タスクに関連する `skills/` ディレクトリの内容を確認してください。
2.  **AIアシスタント:** 正確なコード提案を受けるために、`copilot-instructions.md` がエディタのコンテキストで参照または有効になっていることを確認してください。
